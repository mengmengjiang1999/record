# Designing New Operating Primitives to Improve Fuzzing Performance

仔细一看，这的确是一篇非常重要的文献，对afl++的发展也做出了重要贡献。只不过人家已经做得很好了，我也不知道该做什么。

这篇文章的主题是提高模糊测试（fuzzing）性能的新操作系统原语设计。文章的主要内容包括：

1. **模糊测试的重要性**：模糊测试是一种通过向目标程序注入变异输入来发现软件漏洞的测试技术。它是一种高度实用的方法，并且越来越受欢迎。

2. **现有模糊测试工具的性能瓶颈**：文章指出，现有的模糊测试工具（如AFL）在多核机器上并行运行时，由于文件系统争用和fork()系统调用的可扩展性问题，执行时间会显著增加。

3. **新操作系统原语的提出**：为了提高模糊测试的性能，作者设计并实现了三种新的操作系统原语，这些原语专门针对模糊测试，解决了性能瓶颈问题，并在多核机器上实现了可扩展的性能。

4. **实验结果**：通过实验，作者展示了这些新原语能够显著提高AFL和LibFuzzer在多核机器上的性能。例如，在使用120个核心针对Google的模糊测试套件进行测试时，新原语可以将AFL和LibFuzzer的每秒执行次数分别提高6.1到28.9倍和1.1到735.7倍。

5. **新原语的具体设计**：
   - **snapshot()系统调用**：这是一种新的系统调用，用于高效且可扩展地克隆目标应用程序的新实例，替代了传统的fork()系统调用。
   - **双文件系统服务**：通过在内存文件系统（如tmpfs）和磁盘文件系统（如ext4）之间进行操作，提高了文件操作的性能和可扩展性。
   - **共享内存中的测试用例日志**：帮助模糊测试工具以可扩展和协作的方式共享测试用例执行信息。

6. **对现有模糊测试工具的影响**：文章讨论了这些新原语如何对现有的模糊测试工具（如AFL和LibFuzzer）产生积极影响，并提出了一些改进措施。

7. **相关工作**：文章还讨论了与模糊测试和操作系统可扩展性相关的先前研究。

8. **结论**：文章总结了新原语对提高模糊测试性能的重要性，并指出了未来的研究方向。

整体而言，这篇文章提出了一种通过改进操作系统层面的支持来增强模糊测试性能的方法，这对于发现更复杂的软件漏洞具有重要意义。
